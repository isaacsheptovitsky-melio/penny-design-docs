# Breakpoints

Breakpoints represent how the screen resolution is broken down so we could support the responsiveness of the application across different devices.

### Access the Current Breakpoint

In order to get the current breakpoint the application is rendered in - use the `useBreakpoint` hook.

Link to the doc page - TBD

```tsx
import { useBreakpoint } from '@melio/penny';

const MyComp = () => {
  const { isExtraSmallScreen } = useBreakpoint();

  return isExtraSmallScreen ? <div>I am on mobile!</div> : <div>I am on desktop!</div>;
};
```

### Get a Value Matching a Breakpoint

You can also "ask us" which value should be used for the current breakpoint using the `useBreakpointValue` hook.

Link to the doc page - TBD

```tsx
import { useBreakpointValue } from '@melio/penny';

const MyComp = () => {
  const deviceName = useBreakpointValue({ xs: 'mobile', xl: 'desktop' });

  return <div>I am on {deviceName}!</div>;
};
```
