# Theme

As the design system needs to match itself to the look and feel of the partner we're embedded into, we need the ability to override some of the design tokens of the design-system.<br />
First, let's see how we insert the theme object into the design-system provider.

```tsx
// App.tsx

import { PennyProvider, ThemeOptions } from '@melio/penny';

const theme: ThemeOptions = {
  colors: {
    /*...*/
  },
  borderRadii: {
    /*...*/
  },
  borders: {
    /*...*/
  },
  textStyles: {
    /*...*/
  },
  fonts: {
    /*...*/
  },
  icons: {
    /*...*/
  },
  illustrations: {
    /*...*/
  },
  loader: {
    /*...*/
  },
  blanket: {
    /*...*/
  },
  logos: {
    /*...*/
  },
};

export const App = () => (
  <PennyProvider theme={theme}>
    <TheRestOfMyApp />
  </PennyProvider>
);
```

## Mandatory Values

The theme must have mandatory values for the [Logos](/docs/theme--docs#logos) of which we don't supply any default value.

## What's Included in the Theme

### Tokens

The theme includes design tokens that are used in the components. They can be adjusted according to the partner's theme.

- [Colors](/docs/foundations-colors--docs)
- [Typography](/docs/foundations-typography--docs)
- [Breakpoints](/docs/foundations-breakpoints--docs)
- [Spaces](/docs/foundations-spaces--docs)
- [Border Radius](/docs/foundations-border-radius--docs)
- [Borders](/docs/foundations-borders--docs)
- [Shadows](/docs/foundations-shadows--docs)

### Components

The theme also includes some components that can be adjusted according to the partner's theme.

- [Logos](/docs/foundations-logo--docs)
- [Loader](/docs/foundations-loader--docs)
- [Icons](/docs/foundations-icon--docs)

## Overriding Default Values

### Logos

This is how you can override a partner's [logo component](/docs/foundations-logo--docs).<br />
It will be presented mostly in bars across the pages of the app.<br /><br />
You need to supply _both_ a **dark** logo and a **light** logo.<br />
The **dark** logo will be placed on **light** surface, whereas the **light** logo will be placed on a **dark** surface (as opposed to the idea of "dark/light mode" logo which is the opposite of what we're doing here).

```tsx
// my-theme.tsx

import { ThemeOptions } from '@melio/penny';
import { LightLogoComponent, DarkLogoComponent } from './logos';

export const theme: ThemeOptions = {
  logos: {
    dark: DarkLogoComponent,
    light: LightLogoComponent,
  },
};
```

You can also supply a URL to the logo instead of a JSX element.

```tsx
// my-theme.tsx

import { ThemeOptions } from '@melio/penny';

export const theme: ThemeOptions = {
  logos: {
    dark: 'https://cdn.my-logo.com/dark-logo.png',
    light: 'https://cdn.my-logo.com/light-logo.svg',
  },
};
```

### Colors

We are using design tokens for our colors, here is a simple example of how you can override them.<br />
For the full list of available color tokens, see the [Colors documentation](/docs/foundations-colors--docs)

```tsx
// my-theme.tsx

import { ThemeOptions } from '@melio/penny';

export const theme: ThemeOptions = {
  colors: {
    global: {
      brand: {
        100: '#0276B1',
      },
    },
    semantic: {
      text: {
        primary: '{global.neutral.100}',
      },
    },
    component: {
      button: {
        textSecondary: {
          rest: {
            label: '{semantic.link.primary.hover}',
          },
          hover: {
            label: '{semantic.link.primary.pressed}',
          },
          pressed: {
            label: '{semantic.link.primary.pressed}',
          },
        },
      },
    },
  },
};
```

### Border Radii

We are using design tokens for our border radii, here is a simple example of how you can override them.<br />
For the full list of available border radius tokens, see the [Border radii documentation](/docs/foundations-border-radius--docs)

```tsx
// my-theme.tsx

import { ThemeOptions } from '@melio/penny';

export const theme: ThemeOptions = {
  borderRadii: {
    global: {
      100: '10px',
    },
    semantic: {
      action: {
        default: '{global.full}',
      },
    },
  },
};
```

### Borders

We use design tokens to define border width and style. Here's a simple example of how you can override them in your theme.<br />
For the full list of available border tokens, see the [Borders documentation](/docs/foundations-borders--docs)

```tsx
// my-theme.tsx

import { ThemeOptions } from '@melio/penny';

export const theme: ThemeOptions = {
  borders: {
    global: {
      25: '4px dashed',
    },
    semantic: {
      action: {
        default: '{global.none}',
      },
    },
  },
};
```

### Typography

This is how you can override the typography.<br />
For the full list of available typography, see the [Typography documentation](/docs/foundations-typography--docs)

```tsx
// my-theme.tsx

import { ThemeOptions } from '@melio/penny';

export const theme: ThemeOptions = {
  textStyles: {
    body3Semi: {
      // The available properties to override per typography option:
      fontWeight: string | number;
      textTransform: string;
      letterSpacing: number;
      fontFamily: string;
      fontSize: string | number;
      lineHeight: string | number;
    },
  },
};
```

  fonts: {
    primary: 'My Font Family',

### Loader

This is how you can override the [loader component](/docs/foundations-loader--docs).

```tsx
// my-theme.tsx

import { ThemeOptions } from '@melio/penny';
import { Loader } from './Loader';

export const theme: ThemeOptions = {
  loader: Loader,
};
```

In addition, the loader can come in different colors, so you can use the `color` prop to apply where the custom loader needs to apply the color.<br />
For example:

```tsx
// CustomLoader.tsx

import { ThemeLoaderProps } from '@melio/penny';

export const CustomLoader: VFC<ThemeLoaderProps> = ({ color }) => (
  <Spinner color={color} width="16px" height="16px" thickness="3px" speed="1.3s" />
);
```

This is optional. If you want the loader to be in a static color, you can just ignore the `color` prop.

### Icons

This is how you can override the [icon component](/docs/foundations-icon--docs).

```tsx
// my-theme.tsx

import { ThemeOptions } from '@melio/penny';
import { BankIcon } from './icons';

export const theme: ThemeOptions = {
  icons: {
    bank: BankIcon,
  },
};
```

You can also override the icon per size. Meaning you can override the icon for the `small` and `medium` sizes with different assets.<br />

```tsx
// my-theme.tsx

import { ThemeOptions } from '@melio/penny';
import { BankIconSmall, BankIconMedium } from './icons';

export const theme: ThemeOptions = {
  icons: {
    bank: { small: BankIconSmall, medium: BankIconMedium },
  },
};
```

### Illustrations

This is how you can override the [illustration component](/docs/foundations-illustration--docs).

```tsx
// my-theme.tsx

import { ThemeOptions } from '@melio/penny';
import { BankSuccess } from './illustrations';

export const theme: ThemeOptions = {
  illustrations: {
    bankSuccess: BankSuccess,
  },
};
```

You can also override the illustrations per size. Meaning you can override the illustration for the `small`, `medium` and `large` size with different assets.<br />

```tsx
// my-theme.tsx

import { ThemeOptions } from '@melio/penny';
import { BankSuccessSmall, BankSuccessMedium, BankSuccessLarge } from './illustrations';

export const theme: ThemeOptions = {
  illustrations: {
    bankSuccess: { small: BankSuccessLarge, medium: BankSuccessMedium, large: BankSuccessLarge },
  },
};
```

Another way to load custom illustrations is to upload them to the cloud as `.svg` files and pass the link to the theme:

```tsx
// my-theme.tsx

import { ThemeOptions } from '@melio/penny';

export const theme: ThemeOptions = {
  illustrations: {
    bankSuccess: 'https://my-cdn.couldstorage.com/illustration.svg',
  },
};
```

### Nested Theming

The theme can be nested to apply different appearances to different parts of the tree. This is useful for applying a global appearance and then overriding some parts of it.<br/><br/>
To achieve this, simply wrap the part with another provider:<br/>

- `<PennyProvider />` - use to apply whole theme appearance (tokens, logos, illustrations, icons). <br />
- `<TokensProvider />` - use to apply only tokens appearance (colors, border radii, text styles, spaces and fonts).

You will need to apply the appearance to an element using `cssVarsRoot`. Read [here](https://v2.chakra-ui.com/docs/styled-system/css-variables#attaching-the-css-variables) for more information.

```tsx

// Button with global theme appearance
<Button label="click"/>

// Button with nested theme appearance
<Box id="nested-theme">
  <PennyProvider
    theme={{ logos: {}, colors: { semantic: { action: { primary: { rest: '{global.informative.700}' } } } } }}
    cssVarsRoot="#nested-theme"
  >
    <Button label="click"/>
  </PennyProvider>
</Box>

// Button with nested tokens appearance
<Box id="nested-tokens">
  <TokensProvider colors={{semantic: { action: { primary: { rest: '{global.success.700}' } } } }} cssVarsRoot="#nested-tokens">
    <Button label="click"/>
  </TokensProvider>
</Box>
```
