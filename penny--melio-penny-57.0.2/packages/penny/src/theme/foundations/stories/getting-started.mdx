# Getting Started

Melio has the same design guidelines across its different products.<br />
I guess there's no more need to explain what and why a design system is required and used for, so let's dive in into how to get you started.

## Installation

```
yarn add @melio/penny
// or
npm install @melio/penny
```

## Setting Up

The design-system has a main context that is passed to its components, so you need to wrap your application with it.<br />

```tsx
// App.tsx

import { PennyProvider } from '@melio/penny';
import { theme } from './my-theme';

export const App = () => (
  <PennyProvider theme={theme}>
    <TheRestOfMyApp />
  </PennyProvider>
);
```

<SectionBannerRoot variant="informative">

</SectionBannerRoot>

This will set you up with a design system in our default theme. <br />

Now all you need to do is build your application using our vast component-library and utilities.

## How to Use Storybook

Here are some tips on what you will find on storybook and how to reach it.

### Finding a Component

The main thing you'd do is to look for a component you need for your feature.<br />
Use the sidebar to browse through different sections or use the search bar if you know how the component is called.

### Component Documentation

Once you find your component, you will land on its documentation page.<br />
There you will see, aside from description and usage examples of the component, a control panel with all the properties the component receives.<br />
You can play with the controls and try different component settings.<br />
In the documentation page of a component, the controls only affect the main story (the one above the controls).

### Change Theme

To view how the component looks and feels in different themes, use the toggle at the top bar to switch between themes.<br />
