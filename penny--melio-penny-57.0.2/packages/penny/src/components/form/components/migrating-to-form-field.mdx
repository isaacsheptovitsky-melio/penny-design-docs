# Migration Guide: `Form.*` Components to `FormField`

## Overview

This guide helps you migrate from the deprecated `Form.*` components (e.g., `Form.TextField`, `Form.Switch`, `Form.RadioGroup`) to the new `FormField` component.

## Props Changes

### 1. Render Prop Pattern

The new `FormField` component uses a `render` prop pattern instead of using components directly.

**Old syntax:**
```tsx
import { Form } from '@melio/penny';

<Form.TextField ... />
```

**New syntax:**
```tsx
import { FormField, TextField } from '@melio/penny';

<FormField render={(props) => <TextField {...props} />} />
```

#### 1.1 `fieldContext` (`Combobox` usage)

The `render` function receives a second argument: `fieldContext`. This object contains:
- `labelProps`
- `error`
- `helperText`
- `showOptionalIndicator`

The `fieldContext` must be passed to the `Combobox` component because it internally needs it for mobile view.

```tsx
<FormField
  render={(props, fieldContext) => (
    <Combobox {...props} {...fieldContext} />
  )}
/>
```

<div style={{ marginTop: '60px' }} />

#### 1.2 `isLoading` Prop Handling

The `isLoading` prop is included in the render props passed to your input component. However, some components like `RadioGroup`, `Switch`, `Checkbox` and `SegmentedControl` don't support the `isLoading` prop, so you need to destructure it out. Passing `isLoading` to `RadioGroup` can also trigger console warnings.

```tsx
<FormField
  render={({ isLoading, ...props }) => <RadioGroup {...props} />}
/>
```

#### 1.3 `isViewMode` Prop Handling

The `isViewMode` prop is included in the render props passed to your input component. However, some components like `Checkbox`, `SegmentedControl` and `Switch` don't support the `isViewMode` prop, so you need to destructure it out. Passing `isViewMode` can trigger console warnings.

```tsx
<FormField
  render={({ isViewMode, ...props }) => <Checkbox {...props} />}
/>
```

#### 1.4 `isInvalid` Prop Handling

The `isInvalid` prop is included in the render props passed to your input component. However, some components like `Switch` and `SegmentedControl` don't support the `isInvalid` prop, so you need to destructure it out. Passing `isInvalid` can trigger console warnings.

```tsx
<FormField
  render={({ isInvalid, ...props }) => <Switch {...props} />}
/>
```

### 2. `helperText` Prop

The helper text prop has been simplified from an object to a string.

**Old:**
```tsx
<Form.TextField
  helperTextProps={{ label: 'This will be used as the name on checks' }}
/>
```

**New:**
```tsx
<FormField
  helperText="This will be used as the name on checks"
/>
```

### 3. `isHidden` Prop

The `isHidden` prop has been renamed to `hidden`.

**Old:**
```tsx
<Form.TextField
  isHidden={someCondition}
/>
```

**New:**
```tsx
<FormField
  hidden={someCondition}
/>
```

### 4. `labelProps` prop

For accessibility reasons, `labelProps` should generally be provided. The structure remains the same.

**If no `labelProps` existed before:**
Add `labelProps={{ label: '', isHidden: true }}` to hide the label visually while keeping it for accessibility:

```tsx
<FormField
  labelProps={{ label: '', isHidden: true }}
/>
```

### 5. Other Props

Props that don't belong to the [FormField interface](/docs/form-form-field--docs)  should be passed to the input component via the render prop.

**Old:**
```tsx
<Form.TextField
  placeholder="Enter your name"
/>
```

**New:**
```tsx
<FormField
  render={(props) => (
    <TextField
      {...props}
      placeholder="Enter your name"
    />
  )}
/>
```

## Additional Notes

### Deprecation

The `Form.*` components (e.g., `Form.TextField`, `Form.Switch`, `Form.RadioGroup`) are deprecated and not maintained anymore.

### Documentation

For more details on the `FormField` component, please check out the [FormField docs](/docs/form-form-field--docs).
