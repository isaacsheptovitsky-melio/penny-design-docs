## Usage

- Use tags to categorize or label content clearly.
- Keep tag text short and concise — ideally two words or fewer.
- Use the remove button only when the user needs to dismiss a tag dynamically.

<Callout variant="note">A plain Tag is a static label. Add `onClick` to make it interactive, or `removeButtonProps` to make it removable.</Callout>

## Anatomy

Content is a slot, so you can compose a left element, label, and right element. Tags have a fixed height.

<Canvas of={TagStories.Default} />

### Removable

Pass `removeButtonProps` to render a remove (close) button.

### Clickable

Pass `onClick` to make the whole tag an interactive trigger.

<Canvas of={TagStories.WithLeftElement} />

<Canvas of={TagStories.Disabled} />

### Grouped tags

In a Group, tags space evenly and rearrange when one is removed — the common pattern for applied filters.

<Canvas of={TagStories.GroupOfTags} />

<Callout variant="warning">Don't mix removable and non-removable tags in the same group — separate them to keep the pattern clear.</Callout>
