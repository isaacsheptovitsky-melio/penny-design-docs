## Usage

Use a Container to:

- Create sections or blocks of content.
- Apply consistent spacing and alignment.
- Apply background color and borders to differentiate sections.
- Control dimensions and overflow behavior.

<Callout variant="note">Need an interactive component? Use the Card component instead. The Container is a non-interactive structural element.</Callout>

## Key features

- Flexible padding on all sides (`paddingX`/`paddingY` or per-side `paddingTop`/`paddingBottom`/`paddingLeft`/`paddingRight`).
- Optional borders and background colors.
- Layout control via `width`, `height`, `alignItems`, `justifyContent`.
- Sticky positioning support (`isSticky`), plus `position` / `maxWidth` for finer control.

## Variants

### Background colors

- **default** — transparent.
- **light** — the `semantic.surface.secondary` color.
- **white** — a white background.

<Canvas of={ContainerStories.Backgrounds} />

### Border

`none`, `regular` (1px solid), or `dashed`.

<Canvas of={ContainerStories.Borders} />

## Positioning

Set `position` to control placement — `static` (default), `relative`, `absolute`, `fixed`, or `sticky`. Use `isSticky` as a shorthand for a sticky container (it overrides `position` and `top`). For finer width control, set `maxWidth` (any CSS max-width value).
