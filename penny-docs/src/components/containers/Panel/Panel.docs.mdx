## Usage

- Use for fixed or sticky toolbars, footers, or contextual panels.
- Set `position` to `sticky` (default) for panels that stay visible within scrollable containers.
- Use `fixed` or `absolute` for overlays.
- Adjust the background color to match the use case.

## Variants

### Position

- **Sticky** (default) — sticks within a scrollable container to the top or bottom.
- **Fixed** — stays in the same position relative to the viewport.
- **Absolute** — positioned relative to the nearest positioned ancestor.

### Placement

The panel can stick to any of the four sides of its container.

<Canvas of={PanelStories.Placement} />

<Callout variant="note">The panel animates in with the Slide transition by default — pass a `transitionConfig` (with `in`) to control it.</Callout>

## Use cases

A common use is a batch-action bar: when one or more rows in a table are selected, the panel appears to enable actions on the selection.

## Mobile use

The panel isn't shown on mobile — it's hard to display effectively on small screens.
