## Links

Link typography uses the `standalone` or `inline` variant, with two sizes:

- **Large** (`size="large"`) — **body2** / **body2Semi** · 16px / 24px
- **Medium** (`size="medium"`) — **body3** / **body3Semi** · 14px / 24px _(line-height bumped from 20→24px for a11y click area)_
- **Inline** (`variant="inline"`) — inherits the text style of the surrounding text

The `isBold` prop switches from Regular to SemiBold within the same size. Color is applied via `semantic.link.primary.rest` with a transition to `semantic.link.primary.hover` on hover.

## Naked Buttons

Naked button labels use **body2Semi** (16px/24px) for the large size and **body3Semi** (14px/20px) for the medium size — no dedicated text-style token; they map directly to the body equivalents.

## Recommendations

### Subtitle text

Subtitle text follows an `<h1>` and adds context or elaboration beneath it. Use `body2` in `semantic.text.primary` color.

## Accessibility & DOM structure

Visual hierarchy is not the same as document structure — a style that *looks* like a headline doesn't have to be an `<h1>`.

- Each page has **only one** `<h1>` (its main title); subsequent headings follow a logical `<h2>`, `<h3>`… order.
- Treat typography as the **visual layer** and semantic HTML tags as the **structural layer** — pick the tag from content order, not appearance.
- This keeps screen-reader navigation correct and avoids out-of-order heading levels.
