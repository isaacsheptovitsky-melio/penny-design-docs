## Accessibility & DOM structure

Visual hierarchy is not the same as document structure — a style that *looks* like a headline doesn't have to be an `<h1>`.

- Each page has **only one** `<h1>` (its main title); subsequent headings follow a logical `<h2>`, `<h3>`… order.
- Treat typography as the **visual layer** and semantic HTML tags as the **structural layer** — pick the tag from content order, not appearance.
- This keeps screen-reader navigation correct and avoids out-of-order heading levels.
