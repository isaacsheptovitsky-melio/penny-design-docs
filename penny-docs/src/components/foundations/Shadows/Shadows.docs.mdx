## Usage

- Shadows indicate **elevation**, not decoration.
- Use them meaningfully — when elements need to stand out due to interaction, layering, or context.
- Avoid shadows purely for aesthetics; they should help users understand interface structure.

## Shadow tokens

Each level offers increasing visual weight; pick one based on how elevated an element needs to appear.

<Callout variant="note">Menus and other floating surfaces that open over content use `shadow-500` for a clear floating effect.</Callout>
