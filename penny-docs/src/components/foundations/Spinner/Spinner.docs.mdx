## Usage

Use the Spinner when:

- A page or section is awaiting data.
- A user-initiated action triggers a delay (submitting a form, fetching data).
- An inline element is loading inside a button, input, or container.

<Callout variant="note">The Spinner is meant to sit inside other components, paired with a live region so assistive tech announces the wait — not used standalone.</Callout>

## Types

<Canvas of={SpinnerStories.Variants} />

**Neutral** — The default for most contexts. Use on white or light-grey backgrounds.

**Brand** — Ties the loading moment to brand identity. Use for first-time experiences or screens where brand presence is intentional (e.g. initial app load, payment confirmation).

### Inverse

For dark or colored surfaces where the neutral color would disappear.

<Canvas of={SpinnerStories.Inverse} />

**When to use:** Inside filled buttons, on dark overlays, on colored banners or panels.

## Sizes

<Canvas of={SpinnerStories.Sizes} />

**When to use Small:**

- Inside text fields or inputs (loading a validation state)
- Inside compact action buttons
- In table cells or list items where vertical rhythm is tight

For one-off cases, `size` also accepts a number (px) — e.g. `size={20}` — when neither preset fits.

## States

### Enabled

The spinner animates continuously. Arc color follows the `variant` prop.

### Disabled

When `disabled`, all variants collapse to the same muted palette to signal unavailability — the animation still runs.
