## Usage

Use the Spinner when:

- A page or section is awaiting data.
- A user-initiated action triggers a delay (submitting a form, fetching data).
- An inline element is loading inside a button, input, or container.

<Callout variant="note">The Spinner is meant to sit inside other components, paired with a live region so assistive tech announces the wait — not used standalone.</Callout>

## Types

- **Neutral** (default) — loading on white or light-grey backgrounds; the default for most contexts.
- **Brand** — ties the loading moment to brand identity (initial app load, payment confirmation).

<Canvas of={SpinnerStories.Variants} />

### Inverse

For dark or colored surfaces where the neutral color would disappear — filled buttons, dark overlays, colored banners.

<Canvas of={SpinnerStories.Inverse} />

## Sizes

Use **Small** inside inputs, compact buttons, or tight table cells; **Medium** elsewhere.

<Canvas of={SpinnerStories.Sizes} />

## States

When `disabled`, all types collapse to the same muted palette to signal unavailability — the animation still runs.
