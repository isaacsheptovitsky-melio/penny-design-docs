## Usage

- Use Pagination at the **bottom of a table or list** when the total number of items exceeds the page size (e.g. 25, 50, or 100).
- The label reads `start–end of total` (e.g. *1–25 of 200*), so users always know where they are in the set.
- Drive it from state: keep `currentPage` in the parent and update it from `onPageChange`.
- Controls auto-disable at the boundaries — the previous chevron is off on page 1, the next chevron is off on the final page.

<Callout variant="note">Pagination is a navigation control, not a data layer — it only reports the requested page via `onPageChange`. Fetching or slicing the page of data is the parent's responsibility.</Callout>

## Disabled

Set `isDisabled` to turn off both controls — typically while a page of data is loading.
