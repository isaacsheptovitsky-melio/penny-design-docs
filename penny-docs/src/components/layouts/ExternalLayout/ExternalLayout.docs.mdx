## Usage

- Use External Layout for pages **viewed by your users' customers** — vendors, guest payers, and other unauthenticated visitors.
- Put auxiliary actions like **"Login" or "Help"** in the `header` slot (the top-right area).
- Keep the `content` focused on a **single task** — the content column is capped at 600px so forms and confirmations stay readable.
- Use the `footer` for **legal disclaimers, copyright, or secondary links**.

## Anatomy

The External Layout is made up of four parts:

- **Header** — the area positioned at the top right. Use it for auxiliary actions like "Login" or "Help".
- **Logo** — displays the Melio logo by default, anchored top-left. Can be overridden for partner branding.
- **Content** — the primary workspace, where form inputs, payment details, or confirmation messages live.
- **Footer** — the bottom section, typically legal disclaimers, copyright info, or secondary links.

## Logo

By default the layout renders the **Melio wordmark** in the top-left of the fixed header.

### Override the logo

Pass a custom element to the `logo` prop to replace Melio's branding — use this for **white-labeled experiences** where the page should carry a partner's logo.

<Canvas of={ExternalLayoutStories.OverrideLogo} />

<Callout variant="tip">When creating white-labeled experiences for partners, use the `logo` prop to swap in the partner's logo so the page feels native to their brand.</Callout>

### Hide the logo

Pass an empty element (`logo={<></>}`) to remove the logo entirely.

<Canvas of={ExternalLayoutStories.HideLogo} />

## Loading state

Set `isLoading` to overlay the layout with a centered Loader, preventing interaction during critical background updates.

<Canvas of={ExternalLayoutStories.LoadingState} />

## Mobile usage

<Callout variant="note">The layout is fully responsive. On mobile, the internal padding tightens to maximize screen space, and the content and footer stack vertically while remaining legible on small touchscreens.</Callout>
