## Usage

- Use **Layout** as the top-level container for **every page** in the application.
- Choose a `maxWidth` for the content area to **prevent content from stretching too wide** on large desktop screens.
- Use the `isLoading` state to give **visual feedback** while the page fetches data.
- The Content area is **always centered horizontally** within the container.

<Callout variant="tip">Use narrower widths like `600px` for focused-task pages (such as forms) to improve user focus. Reserve wide widths for dense, data-heavy screens.</Callout>

## Structure

The layout is divided into three distinct slots:

- **Header** — a slot at the top of the page. Can be set to `isSticky` to remain visible while the user scrolls.
- **Content** — the main body where your primary components live. This is the area constrained by `maxWidth`.
- **Footer** — a slot at the bottom of the page. Can also be set to `isSticky`.

## Max width

`maxWidth` controls the maximum width of the centered Content section. It supports the fixed widths `480px`, `600px`, `800px`, `900px`, `1200px`, and `1600px`; `full` to span the entire container; or a custom pixel number.

<Canvas of={LayoutStories.MaxWidth} />

<Callout variant="note">You can pass a custom max width (e.g. `'1140px'`), but avoid it where possible — sticking to the standard widths keeps the layout consistent across the platform.</Callout>

## Background color

Set the background color of the Content area to separate it from surrounding sections.

- **default** — the standard, slightly off-white background for most pages.
- **white** — a pure white background.
- **lightest** — a very light gray, useful for subtle separation between page sections.

<Canvas of={LayoutStories.BackgroundColor} />

<Callout variant="warning">For accessibility, ensure the background you choose provides sufficient contrast with the text on top of it. Avoid combinations that are hard to read.</Callout>

## Sticky header and footer

Set `isSticky` on the header and/or footer to pin them while the Content scrolls — useful for persistent page titles and action footers.

<Canvas of={LayoutStories.StickyHeaderAndFooter} />

## Loading state

In its loading state, a Loader is displayed over the Content area. This blocks interaction until loading is complete.

<Canvas of={LayoutStories.LoadingState} />

## Mobile usage

<Callout variant="note">On mobile, `maxWidth` has little effect — the screen is usually narrower than the available widths. The layout adapts to fit the screen and the content reflows vertically.</Callout>
