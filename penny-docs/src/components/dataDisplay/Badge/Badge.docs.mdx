## Usage

Use a Badge to communicate sync status, vendor verification, an unread count, or other relevant detail layered on top of an element. The `mark` prop holds the overlay; `children` is the element it sits on.

## Variants

### Positioning

The mark can sit at any of the four corners via `placement`.

<Canvas of={BadgeStories.Placement} />

### Custom content

The mark accepts any element — a Counter, a Status Indicator, or a custom logo/icon.

<Canvas of={BadgeStories.CustomContent} />

## Do's and don'ts

- **Do** add a border (`hasBorder`) when the mark's color matches the underlying component, so it stays distinct.
- **Don't** use color alone for critical information — always include a clarifying icon or text.

<Callout variant="note">Ensure sufficient color contrast between the mark and its background, and don't rely on color alone to convey meaning.</Callout>
