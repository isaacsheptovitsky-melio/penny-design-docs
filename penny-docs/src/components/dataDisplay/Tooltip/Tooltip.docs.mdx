## Usage

- Explain **icon-only buttons** or otherwise ambiguous UI elements.
- Provide **extra information** on hover (e.g. "Scheduled to send on May 2").
- Clarify **disabled states or restricted access** (e.g. "Only admins can edit roles").

<Callout variant="tip">Tooltips are for hints, not full explanations. If it needs a sentence or more, consider a popover or modal.</Callout>

## Mobile use

Tooltips do **not** appear on hover on touch devices. For mobile:

- Use alternative UI — help icons, modals, or inline text.
- Don't rely on hover-only interactions; ensure all info is reachable another way.

## Accessibility

- Ensure tooltips can be triggered by both **hover** and **keyboard focus**.
- **Disabled elements** should still expose their tooltip content.
- Keep tooltip text **short, clear, and supportive** — it must never replace essential information.
- Tooltips assist; they should never block task completion or hide important actions.
