## Usage

- **Show status** — warnings, errors, success, or general info.
- **Label content** — users, companies, transactions.
- **Add recognition** — highlight key details without clutter.
- **Use icons** — add logos, avatars, or symbols for clarity.

## Do's and don'ts

- **Do** use concise, clear labels.
- **Do** use icons to add meaning or context.
- **Don't** overuse pills in dense layouts.
- **Don't** use long text strings that break the design.

<Callout variant="note">On mobile, avoid long text to maintain readability.</Callout>

## Accessibility

Don't rely on color alone — use icons and text so the meaning is clear for everyone, including users with color blindness.
