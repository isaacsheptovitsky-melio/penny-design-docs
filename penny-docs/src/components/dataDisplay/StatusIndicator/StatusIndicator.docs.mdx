## Usage

- **Convey system or item status** — communicate the state of an item (e.g. paid, active).
- **Signal notifications** — place on a component like an Icon Button to indicate new, unread activity.

<Callout variant="note">Always provide a textual equivalent (e.g. via a Tooltip) — don't rely on color alone to convey meaning.</Callout>

## Application

To place the indicator on top of another component, wrap it with the **Badge** component.

## Variants

### Status

The `status` prop sets the indicator's color to match its semantic meaning — critical, warning, success, informative, brand, neutral, or secondary.

<Canvas of={StatusIndicatorStories.Statuses} />

### States

Because the indicator often accompanies interactive elements, it supports states consistent with its container.

- **Read-only** — appears dark grey; visible for information but not changeable.
- **Disabled** — muted / de-emphasized.

<Canvas of={StatusIndicatorStories.States} />

### With a label

The indicator can sit beside text.

<Canvas of={StatusIndicatorStories.WithLabel} />
