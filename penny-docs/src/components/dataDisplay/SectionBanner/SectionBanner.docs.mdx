## Usage

Section banners provide key information or alerts within a specific part of a page. Use them to call attention to:

- **Additional information** relevant to the current context.
- **Critical messages** that need to be seen before the user continues.
- **Actions** the user can take to resolve an issue or improve their experience.

<Callout variant="tip">Replace toasts with section banners for short, dismissible system notifications — toasts are transient, easily missed, and often inaccessible to assistive technologies.</Callout>

## Variants

Seven variants set the background, border, and default icon to a semantic intent. Pick based on the **meaning** of the message, not the color.

- **`secondary`** — outline; subtle separation without strong color association.
- **`neutral`** — standard, contextual information.
- **`informative`** — highlights key information and draws attention.
- **`warning`** — potential issues or cautionary information.
- **`critical`** — severe problems or errors that demand immediate action.
- **`success`** — confirms a successful action or positive outcome.
- **`brand`** — a branded feel for general information or promotions.

### When to use an icon

Use an icon when it adds clarity to the message (e.g. status or attention). Avoid icons when the message is already clear and additional visual cues would be redundant.

## Stacking

When placing section banners on top of each other, keep a **12px gap** between them and place the **newest alert on top**.

<Story of={SectionBannerStories.Stacking} />

## Layout

Placement depends on the message:

- **Global placement** — for issues that impact the whole screen or process, position the banner at the top of the screen to demand immediate attention.
- **Local placement** — for an issue scoped to a section, field, or action, position the banner directly above that area.

<Callout variant="tip">For account- or organization-level messages, use the Panel component instead.</Callout>
