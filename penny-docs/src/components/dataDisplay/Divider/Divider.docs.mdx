## Usage

- **Separate sections** — create a clear boundary between distinct groups of information, such as "Vendor details" and "Payment method" in a workflow.

<Callout variant="tip">Use dividers sparingly — overusing them creates visual clutter. Often, generous whitespace is enough to separate related content.</Callout>

## Variants

### Horizontal

The default. A full-width line that separates stacked sections of content.

<Story of={DividerStories.Horizontal} />

### With label

Embed short text within the divider — for example "Or" to divide a group of options. The label renders only on the horizontal variant.

<Story of={DividerStories.WithLabel} />

### Vertical

An upright line used to separate inline elements, such as button groups or key data points in a horizontal summary bar.

<Story of={DividerStories.Vertical} />

<Callout variant="note">The vertical divider takes its height from its container — place it inside a flex row with a defined height.</Callout>
