## Usage

- Use for **non-primary, low-visibility actions** — the naked button draws minimal visual attention.
- It works well in dense UIs alongside stronger button variants to support the hierarchy.
- Supports optional `leftElement` and `rightElement` slots for icons.
- Use `isDisabled` to prevent interaction when an action is temporarily unavailable.

<Callout variant="note">The naked button is ideal for inline actions such as "Learn more", "Edit details", or "View invoice" — where a full button weight would distract from the primary action.</Callout>

## Naked button as link

When clicking the button navigates the user to another route or external page, pass a `link` prop. Penny renders an `<a>` tag under the hood while keeping the visual appearance of a naked button.

```tsx
<NakedButton label="View invoice" link={{ href: '/invoices/123' }} />
<NakedButton label="Open in new tab" link={{ href: 'https://example.com', target: '_blank' }} />
```

<Canvas of={NakedButtonStories.NakedButtonAsLink} />

Even when a link is styled like a button, the underlying HTML must match the function. Screen readers rely on `<button>` vs `<a>` to tell users whether the element triggers an action or navigates. Passing `link` ensures the right element is rendered.

## Accessibility

- **Keyboard**: `Tab` to focus, `Space`/`Enter` to activate.
- **Focus ring**: a 2px offset focus ring renders on `:focus-visible`.
- **Semantics**: renders as `<button>`; passing `link` prop switches to `<a>` so screen readers announce navigation correctly.
- **Disabled**: uses the native `disabled` attribute, removing the button from focus order. For `link` buttons, `aria-disabled` is used instead.
- **Ellipsis**: when `shouldSupportEllipsis` is enabled, the full label is available to screen readers even when visually truncated.
