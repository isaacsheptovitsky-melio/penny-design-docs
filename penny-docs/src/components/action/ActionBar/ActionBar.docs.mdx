## Usage

- Use the Action Bar when a user **selects multiple items** and needs to apply a single action to all of them.
- Only show the bar when **selections are made** — hide it when nothing is selected.
- Display a **clear summary** of the selected items on the left (e.g. a count and a running total).
- Place the **primary action on the right**, with neutral actions (like Cancel) to its left.

## Variants

### Max width

The bar's `maxWidth` constrains its content column. It supports the fixed widths `480px`, `600px`, `800px`, `1200px`, and `1600px`; `full` to span the entire container; or a custom pixel number.

<Canvas of={ActionBarStories.MaxWidth} />

<Callout variant="tip">Align the Action Bar's max width with the Layout grid (e.g. `1600px`) to maintain visual consistency with the page content.</Callout>

### Loading state

Set `isLoading` to replace the summary and actions with the Penny **Loader** and a `loadingText` message — use it while waiting for an async response after the user submits a bulk action.

<Canvas of={ActionBarStories.LoadingState} />

### Background color

The background defaults to `semantic.surface.inverse` (near-black) and accepts any Penny color token — for example `global.brand.700` for a brand-purple surface.

<Canvas of={ActionBarStories.BackgroundColor} />

## Mobile usage

<Callout variant="note">The Action Bar is **not available on mobile** — bulk actions are not supported in mobile views, so the pattern is desktop-only.</Callout>
