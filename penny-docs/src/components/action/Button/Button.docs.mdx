## Usage

- Use for **primary** actions such as submitting forms or initiating key flows. Reserve `secondary` and `tertiary` variants for less prominent, supporting actions.
- Use the `disabled` variant to prevent interactions when an action is unavailable.
- Don't use a `disabled` button to signal invalid form input — let the user submit and respond with a clear error instead.

## Critical button UX pattern

Destructive actions follow a **two-step confirmation flow** to prevent accidental clicks:

1. The initial trigger is a `tertiary` button labeled clearly (e.g. *Delete vendor*).
2. Clicking it opens a confirmation modal whose primary action **is** the `critical` button (e.g. *Delete bill*).

This creates a deliberate pause before any irreversible action.

### Placement

Position the essential action on the right; place secondary or optional actions to its left. Buttons sit horizontally side-by-side — never stacked vertically on desktop.

Surface up to **three buttons per area**. If there are more, consolidate lower-priority actions (Edit, Delete, Download, Share) into a **"More"** menu.

## Button as link

When clicking the button navigates the user to another route or external page, pass a `link` prop. Penny renders an `<a>` tag under the hood while keeping the visual appearance of a button.

```tsx
<Button label="View invoice" link={{ href: '/invoices/123' }} />
<Button label="Open in new tab" link={{ href: 'https://example.com', target: '_blank' }} />
```

<Canvas of={ButtonStories.ButtonAsLink} />

Even when a link is styled like a button (or vice versa), the underlying HTML must match the function. Screen readers rely on `<button>` vs `<a>` to tell users whether the element triggers an action or navigates. Passing `link` ensures the right element is rendered.

## Accessibility

- **Keyboard**: `Tab` to focus, `Space`/`Enter` to activate.
- **Focus ring**: a 2px offset focus ring renders on `:focus-visible`.
- **Semantics**: regular button renders as `<button>`; `link` prop renders as `<a>`.
- **Loading**: `aria-busy` is set when `isLoading`; `loadingText` is announced by screen readers.
- **Disabled**: uses the native `disabled` attribute, removing the button from focus order. For `link` buttons, `aria-disabled` is used instead.
- **Icon-only buttons**: provide `aria-label` when no visible label is present.
