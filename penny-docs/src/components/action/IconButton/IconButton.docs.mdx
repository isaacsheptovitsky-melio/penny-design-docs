## Usage

- Use for actions that can be understood through an **icon alone** — edit, delete, close, expand.
- Always provide an `aria-label` describing the action; there is no visible text label.
- Avoid using for **primary** actions such as form submissions — use a labeled `Button` instead.
- Use `disabled` to prevent interactions when an action is unavailable.

<Callout variant="note">Use icon buttons to declutter dense areas like tables, cards, or toolbars — where labelled buttons would crowd the layout.</Callout>

## Icon button as link

When clicking the button navigates the user to a route or external page, pass a `link` prop. Penny renders an `<a>` tag under the hood while keeping the visual appearance of an icon button.

```tsx
<IconButton icon="chevron-right" aria-label="Open invoice" link={{ href: '/invoices/123' }} />
<IconButton icon="external-link" aria-label="Open in new tab" link={{ href: 'https://example.com', target: '_blank' }} />
```

## Accessibility

- **Keyboard**: `Tab` to focus, `Space`/`Enter` to activate.
- **Focus ring**: a 2px offset focus ring renders on `:focus-visible`.
- **`aria-label` required**: there is no visible text, so every icon button must have an `aria-label` that describes the action (e.g. `"Edit vendor"`, not just `"Edit"`). Prefer action-specific labels over generic ones.
- **Semantics**: renders as `<button>`; passing the `link` prop switches to `<a>` so screen readers announce navigation correctly.
- **Disabled**: uses the native `disabled` attribute, removing the element from the focus order. For link variants, `aria-disabled` is used instead.
- **Minimum target size**: all sizes meet the WCAG 2.2 SC 2.5.8 minimum of 24 × 24 px.
- **Tooltip on disabled**: the tooltip is intentionally preserved on disabled icon buttons so users know the action exists even when unavailable.
